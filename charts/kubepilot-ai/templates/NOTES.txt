1. Reach the API gateway:

   kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "kubepilot.fullname" . }}-api-gateway {{ .Values.apiGateway.service.port }}:{{ .Values.apiGateway.service.port }}

{{- if .Values.webUi.enabled }}

2. Reach the Web UI:

   kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "kubepilot.fullname" . }}-web-ui {{ .Values.webUi.service.port }}:{{ .Values.webUi.service.port }}
   open http://localhost:{{ .Values.webUi.service.port }}
{{- end }}

{{ if not .Values.apiGateway.auth.apiKey -}}
NOTE: apiGateway.auth.apiKey is not set. If you did not create the Secret
"{{ .Values.apiGateway.auth.secretName }}" (key: api_key) out of band, the gateway runs
OPEN (dev only). Set apiGateway.auth.apiKey or create that Secret to enforce X-API-Key.
{{- else -}}
3. Trigger an investigation (X-API-Key required):

   curl -sS -X POST http://localhost:{{ .Values.apiGateway.service.port }}/investigations \
     -H "X-API-Key: <your-key>" -H "Content-Type: application/json" \
     -d '{"query":"why is payment-service failing?","namespace":"prod","service":"payment-service"}'
{{- end }}

{{ if not (gt (len (keys .Values.llm.credentials)) 0) -}}
NOTE: no inline LLM credentials. Cloud providers need Secret "{{ .Values.llm.secretName }}"
(keys: anthropic_api_key / openai_api_key / ...) created out of band, or set
llm.credentials. Air-gapped installs use Ollama/vLLM (no key required).
{{- end }}
